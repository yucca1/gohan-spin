# 要求内容

## 概要

カテゴリ「ramen / ラーメン」を、より汎用的な「noodle / 麺類」に変更する。
これによりラーメンだけでなく、うどん・そば等の麺類をひとつのカテゴリで扱えるようにする。絵文字は 🍜 のまま据え置く。

## 背景

- ユーザーから「うどんの絵文字を追加したい」という要望が発端。
- しかし Unicode に「うどん」専用の絵文字は存在せず、麺類の汁物を表す絵文字は 🍜 のみ。
- 個別カテゴリ（ramen / udon / soba …）を増やすより、🍜 を「麺類」全般のカテゴリに一般化する方が、絵文字の重複もなくシンプル。
- カテゴリの内部キー `iconKey` は localStorage に永続化されるが、**このアプリの利用者はゆっか1人のみ**であり、登録データも少量。
  旧 `'ramen'` データの互換性を担保するマイグレーションは過剰設計（YAGNI）と判断し、**実装しない**。
  旧データは「🍽️ その他」表示になるため、ゆっかがブラウザ操作で手動で「麺類」に付け替える運用とする。

## 実装対象の機能

### 1. カテゴリのリネーム（ramen → noodle / ラーメン → 麺類）
- `IconKey` 型のユニオンメンバー `'ramen'` を `'noodle'` に変更
- `ICON_DEFS` の定義を `key: 'noodle'`, `label: '麺類'` に変更（`emoji: '🍜'`, `order: 2` は維持）
- 画面のカテゴリ選択・一覧で「🍜 麺類」と表示される

## 受け入れ条件

### カテゴリのリネーム
- [ ] カテゴリ選択ドロップダウンに「🍜 麺類」が表示される
- [ ] 新規にお店を「麺類」カテゴリで登録でき、一覧で 🍜 と表示される
- [ ] コードベース内（テスト含む）に `'ramen'` への参照が残っていない
- [ ] 旧 `'ramen'` データを読み込んでもアプリがクラッシュしない（「🍽️ その他」にフォールバックする）

## 成功指標

- `npm test` / `npm run lint` / `npm run typecheck` / `npm run build` がすべて成功する

## スコープ外

以下はこのフェーズでは実装しません:

- 旧 `'ramen'` データのマイグレーション・互換性担保（利用者1人・少量データのため不要。手動で付け替える）
- うどん・そば等の麺類サブカテゴリの追加（noodle に統合するため不要）
- 既存の他カテゴリの再編・リネーム

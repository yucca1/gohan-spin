# 要求内容

## 概要

gohan-spin の最初のP0機能として「お店の登録機能」を実装する。店名とアイコン（カテゴリ）を入力してお店を登録でき、登録結果が一覧に表示され、localStorage に永続化される。

## 背景

- 現状 `src/` は `.gitkeep` のみで、アプリの機能はまだ1つも実装されていない。
- ルーレット（コア体験）を回すには、まず対象となる「お店」を登録できる必要がある。お店の登録は他の全機能（一覧管理・ルーレット・当選演出）の前提となる土台。
- 設計ドキュメント（PRD / 機能設計 / アーキテクチャ / リポジトリ構造）は確定済みで、レイヤード構成（View → Service → Repository）とデータモデル（Shop / IconKey）が定義されている。今回はそれを最初に具現化する。

## 実装対象の機能

### 1. お店の登録（フォーム → Service → Repository → localStorage）
- 店名（テキスト）を入力できる。
- カテゴリ／アイコンを選択できる（MVPは絵文字。暫定7種: 🍔ハンバーガー / 🍜ラーメン / 🍕ピザ / 🍣寿司 / 🍛カレー / ☕カフェ / その他）。
- アイコンは内部的に `IconKey`（キー文字列）で保持し、将来 SVG へ差し替え可能な抽象化にする。
- 登録ボタンでお店を登録でき、登録直後は `enabled = true`（ルーレット対象ON）になる。
- 店名が未入力（trim後0文字）や上限超過（50文字超）の場合は登録できず、理由が画面に表示される。
- お店は最大100件まで登録できる（100件到達時は登録不可・理由表示）。

### 2. 登録済みお店の一覧表示（読み取り専用）
- 登録したお店をカテゴリ順（`IconDef.order`、同カテゴリ内は `createdAt` 昇順）で一覧表示する。
- 登録後にフォームがクリアされ、一覧が再描画される。
- リロード後も一覧が保持される（localStorage 永続化）。
- 店名はユーザー入力のため、DOM へは `textContent` で反映する（XSS対策）。

### 3. 永続化とフォールバック（ShopRepository）
- お店データを localStorage（キー `gohan-spin:shops`）に `{version, shops}` 形式の JSON で保存する。
- 読み込み時に JSON 破損・未存在の場合は空配列で継続し、アプリをクラッシュさせない。
- 書き込み失敗（容量超過等）は `StorageError` で通知する。

## 受け入れ条件

### お店の登録
- [ ] 店名を入力し、アイコンを選んで登録できる。
- [ ] 登録直後のお店は `enabled = true` になる。
- [ ] 店名が trim 後1〜50文字でない場合は登録できず、理由（"店名は1〜50文字で入力してください"）が表示される。
- [ ] `iconKey` が許可値でない場合は登録できず、理由が表示される。
- [ ] お店は最大100件まで。100件到達時は登録不可で理由が表示される。
- [ ] `id` は `crypto.randomUUID()` で採番され、`createdAt` / `updatedAt` が ISO 8601 文字列で設定される。

### 一覧表示
- [ ] 登録済みのお店がカテゴリ順（`IconDef.order` → `createdAt` 昇順）で表示される。
- [ ] 登録後にフォームがクリアされ、一覧が再描画される。
- [ ] リロード後も一覧が保持される。
- [ ] 店名が `textContent` で描画され、`<script>` 等が無害化される。

### 永続化・フォールバック
- [ ] 保存→再読込でお店データが一致する（ラウンドトリップ）。
- [ ] localStorage の値が破損/不正でも、例外を投げず空配列で継続する。

### 品質
- [ ] `npm test` / `npm run lint` / `npm run typecheck` がすべてパスする。
- [ ] ロジック層（Service / Repository / icons）にユニット・統合テストがある。

## 成功指標

- 新規ユーザーが説明なしで最初のお店を1分以内に登録できる（PRD セカンダリーKPI「登録のしやすさ」）。
- ロジック層のテストカバレッジが branches/functions/lines/statements 各80%以上（UI層は除外）。

## スコープ外

以下はこのフェーズでは実装しません（後続の機能で対応）:

- お店の編集・削除・対象（enabled）切替の **UI操作**（PRD機能②「一覧表示・管理」）。
- ルーレット（`RouletteEngine` / `RouletteView`）と当選演出（PRD機能③④）。
- GitHub Pages 配信用の `vite.config.ts`（`base` 設定）。デプロイ準備フェーズで対応。
- レスポンシブの作り込み（2カラム/縦積みの最適化）。登録が使える最小限の見た目に留める。
- 効果音・外部リンク・SVGアイコン化（Post-MVP）。

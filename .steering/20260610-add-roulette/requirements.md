# 要求内容

## 概要

円形ホイール型のルーレット機能（コア体験）を追加する。ルーレット対象（enabled=true）のお店をホイールにランダム配置し、Start→等速回転、Stop→減速＋リーチ演出→当選確定、当選演出（点滅・ズーム・紙吹雪）までの一連フローを実装する。

## 背景

gohan-spinの北極星は「みんなでドキドキする体験」であり、ルーレットと当選演出はPRDでP0(必須)に位置づけられた差別化の核。お店の登録・一覧管理機能は実装済みで、`ShopService.listEnabled()` という供給口も用意済み。残るコア体験を完成させる。

## 実装対象の機能

### 1. ルーレット（円形ホイール）
- 対象（チェックON）のお店だけを円形ホイールにランダム順で等分配置する
- Startボタンで等速回転を開始、Stopボタンで減速しながら停止する
- 上部の針（12時方向）が指した店が当選となる
- 対象0件のときはStartできず、対象を選ぶよう促すメッセージを表示する

### 2. 当選演出
- 減速の最終局面で回転がぐっとスローになるリーチ演出（easeOutQuintイージング）
- 当選店を点滅＋ズームで強調し、アイコン＋店名を画面中央に大きく表示する
- canvas-confettiによる紙吹雪を表示する
- 「もう一度」操作で初期状態に戻り、再度回せる

### 3. レイアウト統合
- デスクトップ: お店管理エリアとルーレットエリアを2カラム横並びに配置
- iPhone（狭幅）: 縦積みに切り替え、ルーレットを主役に大きく配置
- ホイール・針・Start/Stopボタンが画面内に収まり操作できる

## 受け入れ条件

### ルーレット（PRD準拠）
- [ ] ルーレットは円形ホイール型で表示され、上部の針が指した店が当選となる
- [ ] ルーレットにはチェックONのお店だけが表示される
- [ ] ホイール上の並び順は一覧（カテゴリ順）とは独立し、ランダムに配置される
- [ ] Startボタンで回転を開始できる
- [ ] Stopボタンを押すとゆっくり減速しながら停止する
- [ ] 対象店が0件のときはStartできず、対象を選ぶよう促すメッセージを表示する
- [ ] 当選演出の後、再度ルーレットを回せる（リセット操作がある）
- [ ] お店の追加・編集・削除・対象切替がアイドル中のホイールに反映される

### 当選演出（PRD準拠）
- [ ] 減速の最終局面で回転をぐっとスロー＆長くするリーチ演出を行う
- [ ] 当選店を点滅させ、アイコン＋店名を画面中央に大きく表示する
- [ ] 当選時に紙吹雪（canvas-confetti）を表示する

### レイアウト
- [ ] デスクトップ幅で2カラム、狭幅で縦積み（ルーレット主役）に切り替わる

### 品質
- [ ] `RouletteEngine` / イージングのユニットテストがあり、ロジック層カバレッジ80%以上を維持する
- [ ] `RouletteView` の振る舞いユニットテストがある（view層は60%基準）
- [ ] `npm test` / `npm run lint` / `npm run typecheck` / `npm run build` が全て成功する

## 成功指標

- Start→Stop→当選演出の一連フローがエラーなく完走する（PRDのコア体験完走率の土台）
- 回転がCSS transform + requestAnimationFrameで滑らかに動作する（60fps目標）

## スコープ外

以下はこのフェーズでは実装しません:

- 効果音（P1 / Post-MVP）
- お店の外部リンク・SVGアイコン（P1 / Post-MVP）
- Chrome系以外のブラウザ対応（P2）
- GitHub Pages配信設定（vite.config.tsのbase設定）

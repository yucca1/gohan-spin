## 概要

ルーレットの状態（回転中・減速中・当選確定）に合わせて効果音を鳴らし、抽選の高揚感を演出する。音はすべて Web Audio API でプログラム合成し、音声ファイルは使わない。

## 背景

- MVPのルーレットは視覚演出（点滅・ズーム・紙吹雪）のみで、聴覚的な盛り上げがない。
- PRDのPost-MVP機能のうち優先度P1。「今日のごはんを決める」体験をよりエンタメ性の高いものにする。
- 音声ファイルを使わない方針により、アセット管理・ライセンス確認・追加のネットワークロードを不要にする。

## 実装対象の機能

### 1. 回転中のカチカチ音

- Start押下で回転が始まったら、針（12時方向）が区画の境界を通過するたびにカチカチ音を1回鳴らす。
- 見た目の回転と音が同期する（お店の数が多いほどテンポが速くなる）。

### 2. 減速中のドラムロール + カチカチ継続

- Stop押下で減速が始まったら、ドラムロールを流し始める。
- カチカチ音は減速中も継続する（回転が遅くなるにつれ間隔が自然に伸びる）。
- ドラムロールは減速終了（当選確定）で止まる。減速は約9〜10.8秒のため、クレッシェンド（だんだん大きく）で緊張感を高める。

### 3. 当選確定時のファンファーレ

- ホイールが完全停止し当選店が確定した瞬間に、お祝いのファンファーレ音を鳴らす。

### 4. ミュートトグル（初期OFF）

- 音のON/OFFを切り替えるトグルボタン（🔊/🔇）を設置する。
- **初期表示は常にOFF**（音を鳴らしたい人だけONにするオプトイン方式）。
- 設定は保存しない（リロードするとOFFに戻る）。
- 音OFFの間は一切の音を鳴らさない（`AudioContext`も生成しない）。

## 受け入れ条件

### カチカチ音
- [ ] 音ONでStartすると、区画境界の通過ごとにカチカチ音が鳴る
- [ ] 音OFFの間は回転しても音が鳴らない
- [ ] 減速中もカチカチ音が継続し、回転に合わせて間隔が伸びる

### ドラムロール
- [ ] 音ONでStopすると、ドラムロールが鳴り始める
- [ ] 当選確定（完全停止）でドラムロールが止まる
- [ ] 「もう一度」（リセット）でもドラムロールが止まる（鳴りっぱなしにならない）

### ファンファーレ
- [ ] 当選確定の瞬間にファンファーレが1回鳴る

### ミュートトグル
- [ ] 初期表示で音はOFF（🔇表示）になっている
- [ ] トグル押下でON/OFFが切り替わり、表示（🔊/🔇）も切り替わる
- [ ] 回転中・減速中にONへ切り替えても音が鳴り始める（減速中ならドラムロールも開始）

### 品質
- [ ] 既存のテスト（`npm test`）・`npm run lint`・`npm run typecheck`・`npm run build`がすべてパスする
- [ ] jsdom（`AudioContext`が存在しない環境）でもアプリ・テストがクラッシュしない
- [ ] ルーレットの60fpsアニメーションを阻害しない

## 成功指標

- 音ONの状態で Start → Stop → 確定の一連フローにおいて、3種類の音（カチカチ／ドラムロール／ファンファーレ）が状態遷移どおりに鳴る。
- 効果音の追加によって既存機能（抽選・演出・お店管理）の動作・テストに影響が出ない。

## スコープ外

以下はこのフェーズでは実装しません:

- ミュート設定のlocalStorage保存（要件として「保存なし」を選択済み）
- 音量調整スライダー
- 音声ファイル（mp3等）による高品質なサウンド
- BGM（背景音楽）
- 効果音の種類のカスタマイズ設定

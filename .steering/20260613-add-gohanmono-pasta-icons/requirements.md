# 要求内容

## 概要

お店のカテゴリ（`IconKey`）に「丼物」と「パスタ」の2種類を追加する。

## 背景

現状のカテゴリは7種（ハンバーガー/ラーメン/ピザ/寿司/カレー/カフェ/その他）。
牛丼・天丼などの「丼物」や「パスタ」を出すお店を、適切なカテゴリで登録できない。
利用者がより自然にお店を分類できるよう、2カテゴリを追加する。

## 実装対象の機能

### 1. 「ご飯もの」カテゴリの追加
- 牛丼・天丼などの丼物を含む、ご飯ものを表すカテゴリ（キーは `gohanmono`）。
- 完全対応する絵文字はないため、ご飯アイコン（🍚）で代替する。
- ユーザーは登録・編集フォームで「🍚 ご飯もの」を選べるようになる。

### 2. 「パスタ」カテゴリの追加
- パスタ料理を表すカテゴリ。絵文字は 🍝。
- ユーザーは登録・編集フォームで「🍝 パスタ」を選べるようになる。

## 受け入れ条件

### 丼物
- [ ] `IconKey` 型に `gohanmono` が追加されている
- [ ] `ICON_DEFS` に `{ key: 'gohanmono', emoji: '🍚', label: '丼物' }` が定義されている
- [ ] 登録・編集フォームの選択肢に「🍚 丼物」が表示される
- [ ] `isIconKey('gohanmono')` が `true` を返す

### パスタ
- [ ] `IconKey` 型に `pasta` が追加されている
- [ ] `ICON_DEFS` に `{ key: 'pasta', emoji: '🍝', label: 'パスタ' }` が定義されている
- [ ] 登録・編集フォームの選択肢に「🍝 パスタ」が表示される
- [ ] `isIconKey('pasta')` が `true` を返す

### 全体
- [ ] `ICON_DEFS` の `order` が昇順を維持している
- [ ] 既存テスト・型チェック・ビルドがすべて通る

## 成功指標

- 新カテゴリ2種が登録・編集フォーム・一覧・ルーレットで正しく絵文字表示される。
- 既存のお店データ（既存 `iconKey`）に影響を与えない。

## スコープ外

以下はこのフェーズでは実装しません:

- SVGアイコンへの差し替え（P1機能）
- カテゴリの絞り込み・検索機能
- 既存お店データのカテゴリ再分類・マイグレーション

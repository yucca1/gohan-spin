## 1. これは何？（一言で）

**外食するお店をルーレットで決める、ブラウザだけで動くWebアプリ。**
ただ決めるだけでなく、**家族みんなで見てドキドキできる"楽しい演出"** を最大の差別化ポイントにする。

## 2. 解決したい課題

- 近所に外食の選択肢（ハンバーガー、ラーメン等）がたくさんあり、毎回「どこ行こう？」と迷う。
- その「迷う時間」を、家族で盛り上がる楽しい時間に変えたい。

## 3. 対象ユーザー

- 外食によく行く本人＋家族（みんなで画面を見る前提）。
- まずは自分の利用環境（Chrome）で快適に動けばよい。

## 4. 主要機能

### 4-1. お店（選択肢）の登録・管理

- 店名を入力して登録する。
- 店を表す**アイコン**を設定する。
  - アイコンは**主要カテゴリごとにあらかじめ用意**しておき、ユーザーが選んで設定する。
  - 例カテゴリ: 🍔ハンバーガー / 🍜ラーメン / 🍕ピザ / 🍣寿司 / 🍛カレー / 🥗サラダ・カフェ など（最終セットは設計時に確定）。
  - **MVPは絵文字で実装**し、将来的にSVGアイコンセットへ差し替えられる設計にする（アイコンはキー文字列で抽象化）。
- お店は**たくさん登録**できる。
- 各お店に**ルーレット対象チェックボックス**を持たせる。
  - **登録直後はデフォルトでチェックON**（＝ルーレット対象）。
  - チェックを外すとルーレット対象から除外される。
  - チェックONの店だけがルーレットに表示される。
- お店の**編集・削除**ができる。
- **一覧表示**:
  - デフォルトで**カテゴリごとにソート**。
  - **ページングなし**で画面上に全件表示。

### 4-2. ルーレット（コア体験）

- **見た目: 円形ホイール（運命の輪）型**。上部の針が指した店に決定。
- **ルーレットに表示する選択肢の並び順**:
  - チェックONの店だけを表示する。
  - **並び順は一覧表示（カテゴリ順）とは別**にし、**ルーレット上ではランダムに配置**する。
  - 目的: 一覧は「探しやすさ」でカテゴリ順、ルーレットは「意外性・公平感」でランダム、と関心を分ける。
- **Startボタン**: ルーレットがスタート（回転開始）。
- **Stopボタン**: 押すと**ゆっくり減速**しながら停止していく。
- 停止して決まった店を**点滅＋ズーム**で目立たせる。
- **当選演出**:
  - 🎯 **リーチ演出**: 減速の最終局面で回転をぐっとスロー＆長くし、緊張感を最大化（「どっち〜！？」を狙う）。
  - ✨ **点滅＋ズーム**: 当選店を点滅させ、**アイコンと店名をセットで**画面中央に大きく表示する。
    - 理由: 同じアイコンの店が複数（例: ラーメンのアイコンが2店）あることはよくあるため、アイコンだけでは当選店を特定できない。**必ず店名を併記**して一意に伝える。
  - 🎊 **紙吹雪**: お祝いの紙吹雪を散らす（`canvas-confetti` 等のMITライセンスライブラリを想定）。

## 5. 技術方針

- **構成**: Vite + TypeScript。
- **サーバーサイドなし**: ブラウザのみで完結。
- **データ保存**: ブラウザ上に保存。**localStorage を採用**（店データは数十件規模なので十分軽快）。
- **対象ブラウザ**: まずは Chrome で動けばよい。
- **公開**: GitHub Pages。
- **ライセンス**: MIT（ただし利用ライブラリのライセンス影響があれば再検討）。
- **品質**:
  - テストはしっかり書く。
  - prettier ＋ eslint でフォーマット／静的チェックをかける。
- 詳しい設計方針（アーキテクチャ・機能設計）は専用Skillで別途相談する。

## 6. 追加開発項目（MVPの後、今期に開発する／優先度: 一段下）

> ⚠️ これらは「スコープ外（やらない）」ではない。**MVPを作った後、今期内に追加開発していく**項目であり、MVPより優先度が一段下がるだけ。設計時には、これらを後から足せるよう拡張ポイントを確保しておく。

- 各お店の選択肢に**外部リンク**（お店のサイト等）を貼れるようにする。
- アイコンを絵文字から**SVGアイコンセット**へ差し替え。
- 効果音（回転中のカチカチ音、停止時のドラムロール等）。※Chromeの自動再生制限はStartボタン操作起点なら回避可能。
- Chrome以外のブラウザ対応。

## 7. 差別化ポイント（北極星）

> **「決める」ツールではなく「みんなでドキドキする体験」を作る。**
> 演出（リーチ・紙吹雪・点滅ズーム）の楽しさこそが、他の単純なルーレットツールとの差別化。
